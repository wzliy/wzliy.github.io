博客搭建

发表于 2020-03-15

最近一直在研究怎么搭建一个好用的博客，通过网上查找资料，发现可以通过GitHub+hexo来搭建一个免费的个人博客，而且可以通过使用markdown来编写博客，部署和更新也非常方便，弄了几天后终于把环境搭建好了，第一篇博客就记录下我的搭建过程以及踩过的坑。

1.原理介绍

github通过使用github page静态页面来生成一个网页，其本身只是html文件。所以我们可在GitHub上新建一个项目，通过项目中提交html文件，可以实现静态页面的浏览，而我们可以通过hexo来生成html文件，这样，就实现了博客的搭建。

1.环境配置

环境要求：
node.js、git以及github
node.js: hexo是基于nodeJS环境的静态博客，需要用到里面的npm工具；
git: 用于提交代码到github仓库

1.1安装node.js和git

2.初始化安装

2.1 安装hexo

新建一个文件夹，用于存放自己的博客文件，比如我的博客文件都存放在D:\study\program\blog目录下

使用npm安装hexo

$ npm install -g hexo-cli

安装完成后，执行以下命令，Hexo将会在指定文件夹中新建所需要的文件。

$ hexo init
$ npm install

新建完成后，指定文件夹的目录如下：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

_config.yml

网站的配置信息，可在此配置大部分参数

package.json

应用程序依赖包的信息，可自由移除


{
  "name": "hexo-site",
  "version": "0.0.0",
  "private": true,
  "hexo": {
    "version": ""
  },
  "dependencies": {
    "hexo": "^3.8.0",
    "hexo-generator-archive": "^0.1.5",
    "hexo-generator-category": "^0.1.3",
    "hexo-generator-index": "^0.2.1",
    "hexo-generator-tag": "^0.2.0",
    "hexo-renderer-ejs": "^0.3.1",
    "hexo-renderer-stylus": "^0.3.3",
    "hexo-renderer-marked": "^0.3.2",
    "hexo-server": "^0.3.3"
  }
}

scaffolds

模板文件夹，新建文章时，Hexo会根据 scaffolds 来新建文件。

Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

主题 文件夹。Hexo 会根据主题来生成静态页面。

创建主题只要在 themes 文件夹内，新增一个任意名称的文件夹，并修改 _config.yml 内的 theme 设定，即可切换主题。

可在主题商店去下载自己喜爱的主题
